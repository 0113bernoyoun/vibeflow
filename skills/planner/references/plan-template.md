# PLAN.md 산출물 템플릿

아래 형식으로 PLAN.md를 생성합니다. 모든 섹션은 필수입니다.

```markdown
# PLAN: {작업 제목}

## Context
{작업 배경 및 목적}

## Assumptions
{Step C에서 확인된 전제 조건들. 이 전제가 틀리면 PLAN 전체를 재검토해야 함}

| 전제 | 내용 | 확인 방법 |
|------|------|-----------|
| {전제명} | {구체적 내용} | {사용자 확인/코드 확인} |

## Architecture
{영향을 받는 클래스 및 레이어 (Service, DTO, Repository 등)}

## Existing Test Baseline

{영향 범위에 포함되는 도메인/모듈의 기존 테스트 현황 및 품질 평가}

| 테스트 파일 | 테스트 메서드 수 | 실행 결과 | 품질 등급 | 처리 방침 | 사유 |
|---|---|---|---|---|---|
| {파일 경로} | {N개} | PASS/FAIL | 양호/빈약/잘못됨 | 보호/보강/수정/삭제 | {판단 근거} |

**기존 테스트 총 수**: {N}개
**보호 대상**: {M}개 (양호 등급)
**보강/수정 대상**: {K}개 (빈약/잘못됨 등급)

## Behaviors (순서대로)

### Behavior 1: {이름}
- 설명: ...
- 입력: ...
- 기대 결과: ...
- 예외 상황: ...
- 영향 범위: ...
- depends_on: {없음 또는 [N] 한 줄 전제 요약}
- testability: {assert 방법}
- mock 전략: {Mock 대상 또는 "없음"}
- size 판정: {단일 입력-결과 대응 ✓}
- 기존 테스트: {처리 방침 또는 "없음"}

### Behavior 2: {이름}
...

## Logical Harness
{behavior 간 흐름이 연결되는지 확인하는 체크리스트}

- [ ] Behavior 1의 출력이 Behavior 2의 입력으로 자연스럽게 이어지는가?
- [ ] 모든 예외 상황에 대한 behavior가 포함되어 있는가?
- [ ] 전체 흐름에서 누락된 단계가 없는가?
- [ ] Assumptions 섹션의 전제가 모든 behavior에 일관되게 적용되었는가?
```
