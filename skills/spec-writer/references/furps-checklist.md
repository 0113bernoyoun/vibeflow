# FURPS+ 비기능 요구사항 체크리스트

스펙 작성 완료 후, 아래 체크리스트로 비기능 요구사항 누락을 점검합니다.
해당 스펙에 관련 없는 항목은 건너뜁니다. 관련 있는데 누락된 항목은 사용자에게 보고합니다.

## Functionality (기능성)

- [ ] **보안**: 인증/인가 요구사항이 정의되었는가?
- [ ] **감사 로깅**: 누가, 언제, 무엇을 했는지 추적 가능한가?
- [ ] **데이터 무결성**: 트랜잭션 범위, 정합성 규칙이 정의되었는가?

## Usability (사용성)

- [ ] **접근성**: 장애인/고령자 접근성 요구사항이 있는가?
- [ ] **에러 메시지**: 사용자에게 보여줄 에러 메시지가 정의되었는가?
- [ ] **국제화/다국어**: 다국어 지원이 필요한가?

## Reliability (신뢰성)

- [ ] **장애 복구**: 장애 발생 시 복구 절차가 정의되었는가?
- [ ] **데이터 백업**: 데이터 손실 방지 요구사항이 있는가?
- [ ] **가용성 목표**: SLA/uptime 목표가 있는가?

## Performance (성능)

- [ ] **응답 시간**: P95 응답 시간 등 구체적 수치가 정의되었는가?
- [ ] **처리량**: 초당/분당 처리해야 할 요청 수가 있는가?
- [ ] **동시 사용자**: 동시 접속자 수 요구사항이 있는가?
- [ ] **데이터 규모**: 예상 데이터 양과 증가율이 파악되었는가?

## Supportability (지원 가능성)

- [ ] **모니터링**: 어떤 메트릭을 모니터링해야 하는가?
- [ ] **로깅**: 디버깅을 위해 어떤 정보를 로깅해야 하는가?
- [ ] **알림**: 어떤 상황에서 알림을 보내야 하는가?
- [ ] **배포**: 배포 전략 (무중단, 카나리 등)이 있는가?

## + (추가 제약)

- [ ] **법규/규정**: 개인정보보호법, 관련 법규 준수 사항이 있는가?
- [ ] **호환성**: 지원해야 할 브라우저/OS/기기 목록이 있는가?
- [ ] **외부 의존성**: 외부 API/서비스의 SLA, 장애 대응이 고려되었는가?
